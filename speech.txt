но сперва я бы хотел рассказать об архитектуре проекта "Планирование бюджета"
на этой схемк представлена упращенная диаграмма компонентов проекта
само приложение работает в сервере приложений wildfly, где расположены следующие компоненты
приложение (application) - это компонент для непосредственного взаимоодействия с пользователем
справочники - компонент для работы со справочной информацией, а также ее синхронизации с НСИ по средством SOAP-протокола
ОБАС (обоснование бюджетных ассигнований) и РРО - одни из основных предметных блоков системы
к каждому из этих компонентов подключены модули операций, процессов на языке kotlin в виде отдельных jar файлов
каждый компонент работает с modeshape (документной БД), данные которой сохранены в postgres
отдельно следует отметить вызов api проекта бюджта для экспорта/импорта данных из блока ОБАС
регистры - способ представления информации данных предметных блоков в плоском виде в potgres
